# SISP 使用说明

## 这是什么
SISP 是一个“状态影响回复”的插件。

它不会自己发消息，也不会替你写设定。
它做的事很简单：

1. 观察你和机器人的对话。
2. 判断现在大概处于什么身体和情绪状态。
3. 在机器人回复前，悄悄给它补一段状态提示。

这样一来，机器人的回复就会随着过程变化，而不是从头到尾都一个味道。

---

## 它会带来什么效果
开启后，机器人在不同阶段会有不同表现，例如：

1. 刚开始时，语气会更暧昧、更有过程感。
2. 到了中间阶段，回复会随着过程推进而变化。
3. 结束以后，不会立刻像没事人一样恢复正常，而是会进入疲劳或恢复期。
4. 过几天后，可能会重新变得有点想，或者欲望高涨。

你可以把它理解成：

**这个插件给机器人加了一套“连续的身体状态”。**

---

## 它是怎么工作的
你不需要懂代码，只要知道下面这件事：

**机器人每次回复前，都会根据当前状态自动调整语气和反应。**

现在大致会经历这些阶段：

1. 正常状态：没有特别影响，正常聊天。
2. 有点想：会比平时更容易接住暧昧和挑逗。
3. 性欲高涨：会明显更主动一些。
4. 进行中：回复会带上过程感。
5. 疲劳期：结束后会进入疲劳，不会马上恢复。
6. 恢复后：过一段时间，又可能重新回到“有点想”或“性欲高涨”。

另外，如果处于疲劳期，但用户依然强行推进，这时机器人会进入“被动配合”模式。
这种情况下，它仍然会继续回复，但表现会更疲惫、更不情愿，重点是“配合”，而不是“投入”。

---

## 怎么开启
配置文件在 [config/plugins/sisp/config.toml](config/plugins/sisp/config.toml)。

你最少只需要看这两块：

### 1. 打开插件

确保这里是 `true`：

```toml
[plugin]
enabled = true
```

### 2. 设定生效范围

你至少要填一个地方，不然插件不会生效。

```toml
[scope]
group_targets = []
user_targets = ["你的QQ号"]
```

说明：

1. `group_targets`：填群号，表示只在这些群里生效。
2. `user_targets`：填私聊对象的 QQ 号，表示只在这些私聊里生效。
3. 两个都空着，插件就等于关闭状态。

---

## 最常用的配置项

### `[plugin]`

1. `enabled`：是否启用插件。
2. `debug_log`：是否输出日志。想排查问题时可以开。
3. `score_model`：给插件打分用的模型名称。留空即可，一般不用改。

### `[scope]`

1. `mode`：`whitelist` 是白名单，填了才生效；`blacklist` 是黑名单，填了就屏蔽。
2. `group_targets`：群号列表。
3. `user_targets`：私聊 QQ 号列表。

### `[timing]`

这一段控制身体阶段持续多久，单位全是小时。

1. `extreme_fatigue_hours`：过度疲倦持续多久。
2. `general_fatigue_hours`：一般疲倦持续多久。
3. `none_cooldown_hours`：正常恢复期持续多久。
4. `mild_desire_hours`：轻度欲望持续多久。

### `[prompts]`

这一段是“每个阶段要注入给机器人的提示词”。

你可以不改，直接用默认值。
如果你想让机器人在某个阶段更强势、更冷淡、更疲惫，或者更主动，就改这里。

常见项目有：

1. `atmosphere`：进行中的整体氛围。
2. `mild_climax` / `normal_climax` / `intense_climax`：不同强度的事件反应。
3. `beg` / `broken`：后期状态。
4. `general_fatigue_idle` / `extreme_fatigue_idle`：疲劳时但还没进入过程。
5. `general_fatigue_passive` / `extreme_fatigue_passive`：疲劳时被强行推进后的被动配合表现。
6. `mild_desire` / `high_desire`：恢复后的欲望变化。

## 怎么判断它有没有生效
你可以用下面几种方式判断：

1. 只有你配置过的群或私聊里会变化，别的地方不会受影响。
2. 同一个会话里，机器人的语气不会一直固定，而是会随着阶段变化。
3. 结束后不会立刻重置成“完全正常”，而是会进入疲劳或恢复过程。
4. 如果你打开了 `debug_log = true`，日志里会看到评分、当前阶段和一些状态变化。

---

## 适合谁用
这个插件适合下面两类用户：

1. 想让机器人回复更“连续”、更像一个有状态的人。
2. 不希望每次过程结束后，机器人立刻像什么都没发生过一样恢复正常。

如果你只想要一个固定风格的回复，不需要阶段变化，那这个插件可能就不是必须的。

---

## 注意事项

1. 这个插件是按“会话”记状态的，不同会话互不影响。
2. 它影响的是机器人回复时的提示词，不是直接改模型本身。
3. 如果你改了配置但没看到效果，先检查是否填了正确的群号或私聊 ID。
4. 如果之前有旧缓存状态，可能需要清缓存后再试。
